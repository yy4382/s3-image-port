S3 Image Port 可以像 iPhone 或 Mac 上的「照片」App 一样，展示带动态播放效果的
Apple **实况照片（Live Photo）**。本页介绍实况照片到底是什么、如何上传才能让面板
识别它，以及配对在背后是如何完成的。

## 什么是实况照片？

实况照片**不是单个文件**。当 iPhone 拍摄一张实况照片时，它会保存两个相互关联的
独立文件：

1. 一张**静态图片**——你看到的那一帧画面，通常是 `IMG_0001.HEIC`（或 `.JPG`）。
2. 一段**动态视频**——记录动作与声音的短片，即 `IMG_0001.MOV`。

把它们联系在一起的唯一线索，就是共享的**文件基名（basename）**：`IMG_0001`。
「照片」App 在内部维护着这层关系，但一旦你把文件导出到电脑、或直接导出到 S3，它们
就变成了两个普通文件。S3 Image Port 正是依据这个共享的名字把它们重新配对起来。

## 如何上传实况照片

<Callout type="warning">
  请在**同一批次中同时上传两个文件**。静态图片和动态视频必须一起加入上传列表，
  面板才能将它们配对。
</Callout>

### 1. 从设备导出两个文件

从「照片」App 或 iCloud 导出时，请确保获取的是**未经修改的原始文件**，这样才能
同时得到图片和 `.MOV` 文件。隔空投送（AirDrop）和「导出未修改的原件」都会保留
这对文件。

### 2. 保持文件名一致

两个文件必须共享相同的基名。扩展名会被忽略，且匹配**不区分大小写**，因此下面这些
都能正确配对：

- `IMG_0001.HEIC` + `IMG_0001.MOV`
- `IMG_0001.JPG` + `img_0001.mov`
- `vacation.png` + `vacation.mp4`

如果你重命名了其中一个，记得把另一个也改成相同的名字。

### 3. 一次性把两个文件拖入上传区

将**两个文件一起**选择或拖入上传区域。当它们处于同一批次且文件名匹配时，面板会
自动把它们关联起来——动态视频会以与静态图片相同的键基名（key base）存储，因此
下次图库加载时它们仍然保持配对。

上传完成后，图库会显示一个带 **LIVE** 标记的单一项目。打开它并按住（或点击）即可
播放动态效果。

## 支持的文件类型

只有当批次中包含**一张静态图片和一段动态视频**、且二者共享基名时，才会形成一对
实况照片。

| 角色     | 支持的扩展名                                                |
| -------- | ----------------------------------------------------------- |
| 静态图片 | `jpg`、`jpeg`、`png`、`webp`、`avif`、`gif`、`heic`、`heif` |
| 动态视频 | `mov`、`mp4`、`m4v`                                         |

<Callout>
  当浏览器无法原生解码 HEIC/HEIF 静态图片时，面板会自动将其转换为浏览器可绘制的
  格式，因此即便在不支持 HEIC 的浏览器上，静态图片及其实况播放都能正常显示。
</Callout>

## 系统如何识别实况照片

配对会在两个时刻发生，二者都依赖共享的基名。

**上传时。** 当一个批次被加入时，每个文件会被归类为图片或视频（依据其 MIME 类型，
若没有则回退到扩展名）。共享基名、且恰好包含一张图片和一段视频的文件会被视为一对。
随后动态视频会以**与静态图片相同的键、仅更换扩展名**的方式保存（例如 `…/abc.jpg`
和 `…/abc.mov`）。这样就保证它们在你的存储桶中共享同一个键基名。如果静态图片在
上传过程中被压缩或转换，动态视频的键会跟随静态图片最终的名字，从而保持配对。

<Callout type="warning">
  如果同一个基名下有多张图片或多段视频，只有各自的第一个会被配对；多出来的文件会
  作为独立文件上传。
</Callout>

**列出时。** 当图库加载你的存储桶时，对象会按键基名（去掉扩展名的路径）分组。只有
当某个基名**同时**拥有静态图片和动态视频时，它才会被算作实况照片。此时：

- 裸的 `.mov` 会从网格中**隐藏**——否则它会显示为一个无法查看的损坏项目。
- 静态图片会带着 **LIVE** 标记展示，其配对的视频会在需要时按需加载，用于按住播放。

没有匹配静态图片的视频会保持原样，作为普通文件列出。

## 故障排查

- **没有出现 LIVE 标记。** 请确认两个文件共享完全相同的基名，且视频使用了受支持的
  扩展名（`mov`、`mp4`、`m4v`）。如果你在不同批次中分别上传、生成了不同的名字，
  它们就不会共享键基名——请把它们重新一起上传。
- **动态效果无法播放。** 浏览器必须能够解码该视频。iPhone 的 `.mov`（HEVC）可以在
  Safari 和大多数现代浏览器中播放；如果失败，面板会回退到仅显示静态图片。
